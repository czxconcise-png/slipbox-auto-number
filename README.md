# Slip Box Auto Number

一个给 Obsidian 卡片盒自动补编号的小插件。它适合用一个 Markdown 大纲文件维护卢曼式卡片盒的人：你先把新笔记按缩进插入到卡片盒树里，插件根据当前位置自动推断编号，再把链接或笔记名改成 `编号 + 标题` 的形式。

## 它解决什么问题

在 Folgezettel / 卢曼笔记法里，新卡片通常不是简单追加到末尾，而是插入到某张旧卡片之后，或者成为某张卡片的子卡片。手动编号时，你需要反复查看上下文：

- 这张新卡片是同级兄弟，还是子卡片？
- 上一张同级卡片的编号是什么？
- 新卡片应该叫 `25.1a`、`25.1b`，还是 `25.1a1`？
- 如果一次插入好几张无编号卡片，后面的编号还要基于前面新生成的编号继续推断。

这个插件把这一步自动化。你只需要把无编号链接放到正确位置，插件会弹窗告诉你准备改成什么编号，确认后再应用。

## 核心功能

- 在指定的 Slip Box Markdown 文件中检测无编号 wikilink。
- 根据列表缩进和上下文推断编号。
- 支持一次插入多条无编号链接，并在同一个弹窗里批量确认。
- 支持空链接：默认只把 `[[标题]]` 改成 `[[编号 标题]]`，不强制创建 md 文件。
- 支持已有笔记：如果链接能解析到真实笔记，会通过 Obsidian 的文件管理器重命名该笔记。
- 保留手动触发命令，自动检测没弹出时也可以主动扫描。
- 不会重排已有编号，已有编号会被视为稳定结构。

## 示例

假设你在卡片盒里有一组关于“为什么人会拖延”的笔记：

```markdown
- [[25.1 拖延不是时间管理问题]]
  - [[25.1a 拖延常常来自任务太模糊]]
```

后来你读书时又写了三张新卡片，想把它们插到这组笔记下面：

```markdown
- [[25.1 拖延不是时间管理问题]]
  - [[25.1a 拖延常常来自任务太模糊]]
  - [[把任务拆小可以降低启动阻力]]
    - [[两分钟版本让大任务变得可开始]]
  - [[拖延时先处理情绪而不是安排日程]]
```

插件会批量建议：

```markdown
[[把任务拆小可以降低启动阻力]] -> [[25.1b 把任务拆小可以降低启动阻力]]
[[两分钟版本让大任务变得可开始]] -> [[25.1b1 两分钟版本让大任务变得可开始]]
[[拖延时先处理情绪而不是安排日程]] -> [[25.1c 拖延时先处理情绪而不是安排日程]]
```

确认后，卡片盒会变成：

```markdown
- [[25.1 拖延不是时间管理问题]]
  - [[25.1a 拖延常常来自任务太模糊]]
  - [[25.1b 把任务拆小可以降低启动阻力]]
    - [[25.1b1 两分钟版本让大任务变得可开始]]
  - [[25.1c 拖延时先处理情绪而不是安排日程]]
```

如果这些链接已经对应真实笔记，插件会同时重命名笔记文件；如果它们还是空链接，默认只修改卡片盒里的链接文本。

## 安装

这是一个本地 Obsidian 插件。把仓库里的文件放到你的 vault 插件目录：

```text
你的Vault/.obsidian/plugins/slipbox-auto-number/
```

至少需要这些文件：

```text
manifest.json
main.js
styles.css
```

然后在 Obsidian 里打开：

```text
设置 -> 第三方插件 -> Slip Box Auto Number
```

如果你刚复制完文件但 Obsidian 没刷新，可以关闭再开启插件，或重启 Obsidian。

## 配置

进入插件设置后，添加你的 Slip Box 文件路径。路径使用 vault 相对路径，例如：

```text
Slip Box/Life Slip Box.md
```

一个 Slip Box 位置就是一个 Markdown 文件。插件只会在这些配置过的文件里检测无编号链接。

全局设置包括：

- `自动检测`：编辑 Slip Box 文件时，自动发现无编号链接并弹出确认。
- `空链接处理方式`：当链接还没有对应笔记时，默认只给链接补编号，不创建笔记。

## 使用方式

正常工作流：

1. 先写好笔记，或者先在卡片盒里写一个空链接。
2. 在 Slip Box 文件中，把 `[[中文标题]]` 放到正确的大纲位置。
3. 调整缩进，让它成为兄弟卡片或子卡片。
4. 等插件自动弹出批量确认框，或手动运行扫描命令。
5. 检查弹窗里的编号建议，确认后应用。

可用命令：

- `给当前无编号链接编号`：只处理光标当前行。
- `扫描当前卡片盒无编号链接`：扫描当前 Slip Box 文件里的所有无编号链接。
- `清除自动检测忽略记录`：如果你取消过一次弹窗，想让同一组链接再次触发自动检测，可以运行这个命令。

右键菜单和左侧 ribbon 按钮也可以触发扫描。

## 编号规则

插件主要根据缩进和附近已有编号判断：

- 顶层卡片会沿用顶层编号序列。
- 有上一张同级卡片时，新卡片作为它的下一个兄弟。
- 没有上一张同级卡片、但有父卡片时，新卡片作为父卡片的第一个子卡片。
- 一次批量处理时，前一条新编号会立刻进入临时上下文，供后一条继续推断。

例如：

```markdown
- [[5.1 Root]]
  - [[5.1a Existing]]
  - [[New A]]
  - [[New B]]
```

会得到：

```text
New A -> 5.1b New A
New B -> 5.1c New B
```

## 注意事项

- 插件不会替你判断卡片内容应该放在哪里；它只根据你已经摆好的位置推断编号。
- 插件不会重新编号已有卡片。
- 如果同一个无编号链接能解析到多个同名笔记，插件会跳过并提示你先消除歧义。
- 如果目标编号已经存在，插件会跳过该条，避免覆盖已有笔记。
- 默认空链接模式不会创建 md 文件，只会修改卡片盒里的链接文本。
